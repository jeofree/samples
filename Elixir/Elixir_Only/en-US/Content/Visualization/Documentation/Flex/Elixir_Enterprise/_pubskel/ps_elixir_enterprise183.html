Accessibility for organization charts

The OrgChart component has built-in support for accessibility: it automates the common accessibility practices of providing text equivalents and promoting keyboard access.
The accessibility implementation class of an OrgChart component is OrgChartAccImpl.
The OrgChart component is exposed through an outline role of MSAA (Microsoft Active Accessibility).
The label of an event exposed to a screen reader is accessible through the itemToLabel() method. By default this method returns the value of the name field of the data item. You can customize the label by providing a custom label function in the labelFunction property.
The user of the orgnaization chart can navigate through the displayed items with the keyboard as described in User interaction with organization charts.
For more information on accessibility, see Creating Accessible Applications in the Advanced Flex Programming Adobe documentation.
