Structure of the graph layout API

The IBM® graph layout API is composed of:
  • The generic graph layout package
  • The layout algorithm packages

The generic graph layout package

IBM® ILOG Elixir provides a high-level, generic framework for the graph layout services: com.ibm.ilog.diagram.graphlayout

The layout algorithm packages

  • com.ibm.ilog.diagram.graphlayout.grid: A layout algorithm that arranges the disconnected nodes of a graph in rows, in columns, or in the cells of a grid.
  • com.ibm.ilog.diagram.graphlayout.hierarchical: A layout algorithm that arranges nodes in horizontal or vertical levels such that the links flow in a uniform direction.
  • com.ibm.ilog.diagram.graphlayout.longlink: A layout algorithm that reshapes the long orthogonal links of a graph without moving the nodes.
  • com.ibm.ilog.diagram.graphlayout.shortlink: A layout algorithm that reshapes the short links of a graph without moving the nodes.
  • com.ibm.ilog.diagram.graphlayout.random: A layout algorithm that moves the nodes of the graph at randomly computed positions inside an user-defined region.
  • com.ibm.ilog.diagram.graphlayout.tree: A layout algorithm that arranges the nodes of a tree horizontally or vertically, starting from the root of the tree. A radial layout mode allows you to arrange the nodes of a tree on concentric circles around the root of the tree.
  • com.ibm.ilog.diagram.graphlayout.forcedirected: A layout algorithm that can be used to lay out any type of graph and allows you to specify the length of the links.
