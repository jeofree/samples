Animating changes

The OrgChart component manages two different kinds of animation that can be run at the same time:
  • View animation
  • Item animation

View animation

View animation consists of changing the visible area of the chart smoothly, resulting in panning and/or zooming the view. It is triggered by the view manipulation API, see View manipulation API.
The duration of the view animation is defined by the animationDuration style property and the motion is defined by the easingFunction style property.
When the visible bounds are changed during view animation, a visibleBoundsChange event is dispatched. The current visible bounds are accessible through the visibleBounds property.

Item animation

Item animation consists of moving each item renderer smoothly when the OrgChart layout has changed.
Item animation can be triggered by any of the following:
  • A node has been expanded or collapsed (see Opening and closing nodes).
  • A node has been added or removed as the child of another node.
  • A node layout has changed.
  • A node assistant state has changed.
The duration of the item animation is defined by the itemAnimationDuration style property and the motion is defined by the itemEasingFunction style property.
When item animation is in progress, an itemsAnimated event is dispatched.
The current bounds of the item renderer associated with a data item are accessible using the getItemBounds method.
If the animation Boolean parameter of this method is set to true, the current bounds are returned; otherwise the bounds at the end of animation are returned.
The bounds of the subtree of an item are returned by the getItemSubtreeBounds method.

Determining the end of animation

When animation is in progress, animation events of type OrgChartEvent are dispatched. The adjusting property of the event is set to true when animation is in progress and then to false on the final step of animation.
