Using map control skins

Defining a skin is the best way to fully control the appearance and functionality of a Map component.
You define a skin to be used by the Map component by setting the skinClass property to the appropriate skin class:
<ContinentalEuropeMap skinClass=”MySkin”/>
The skin class called MySkin contains all the groups that the Map component will display. This skin must define the mandatory skin part as defined in the MapBase class. These mandatory parts are the MapGroup, MapFeatureGroup and the Scroller. The MapGroup is a Group holding all the groups or layers that the Map component will display. This MapGroup is also used by the Scroller as the viewport to zoom and pan the map. The Scroller allows map navigation such as zooming and panning. The Scroller also manages the scrollbars around the map.
The MapFeatureGroup is the Group that holds the MapFeatures, which are the geographic entities that the map component will display. These geographic entities are rendered as polygons to display the shapes of a country or a state.
Defining the MapGroup and MapFeatureGroup
You define the MapGroup and MapFeatureGroup inside an mxml file describing the skin:
<!-- The layer container -->
  <ibm:MapGroup id="mapGroup">
    <!-- the layer holding map features -->
    <ibm:MapFeatureGroup id="featureGroup"/>
  </ibm:MapGroup>
The id of the MapGroup must be mapGroup and the id the MapFeatureGroup must be mapFeatureGroup, in order to match the declarations made into the Maps component.
Using the Scroller control
Adding a Scroller control in the skin will allow the navigation through the map. To define the scroller in the skin class you declare it as follows:
<!--- The Scroller component to navigate into 
the map and add scroll bars to the map. -->
  <ibm:Scroller panModifierKey="ctrl" 
                wheelZoomModifierKey="ctrl"
                id="scroller" >
    <!-- The layer container -->
    <ibm:MapGroup id="mapGroup">
      <!-- the layer holding map features -->
      <ibm:MapFeatureGroup id="featureGroup"/>
    </ibm:MapGroup>
  </ibm:Scroller>
The id of the scroller must be ‘scroller’ in order to match the declarations made in the Maps component.
