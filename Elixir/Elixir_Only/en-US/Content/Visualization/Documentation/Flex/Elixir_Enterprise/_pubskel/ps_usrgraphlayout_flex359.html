Graph layout: using the API

In an application that does not work with diagrams or graphs (Graph), operations such as attaching or detaching a graph layout instance must be performed explicitly.

The base class: GraphLayout

The GraphLayoutclass is the base class for all layout algorithms. This class is an abstract class and cannot be used directly. You must use one of its subclasses: HierarchicalLayout, TreeLayout, ForceDirectedLayout, LongLinkLayout, ShortLinkLayout, RandomLayout, GridLayout. You can also create your own subclasses to implement other layout algorithms. See Defining your own type of layout.
Although only subclasses of GraphLayout are directly used to obtain the layouts, you need to learn about this class because it contains methods that are inherited (or overridden) by the subclasses. You will need to understand it to create your own subclasses.
