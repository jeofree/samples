Customizing node and link renderer skins

Node and link renderers are implemented as skinnable components, by Node and Link classes. As skinnable components, these classes are host components to the skin classes, which will effectively provide the graphic representation of the object in a Diagram component. These classes have default associated skins, which are implemented by NodeSkin and LinkSkin present in the com.ibm.ilog.diagram.skins.spark package. Skins can be created and applied to nodes and links using CSS, MXML or ActionScript. See LINKTO [Using the Node Class] for information on how to configure the Node class and its skin. See LINKTO [Using the Link class] for information on how to configure the Link class and its skin. When you create skins, you can access properties of the host component from inside the skin by using the strongly typed hostComponent property as a reference to the component. For example: 
<s:Label  
  id="labelElement"  
  textAlign="center" 
  alignmentBaseline="useDominantBaseline"
  verticalAlign="middle"
  lineBreak="toFit" 
  left="10" right="10" 
  top="10" bottom="10"
  text="{hostComponent.label}"
  horizontalCenter="0" verticalCenter="1"
/>
As the host component is a Renderer (an instance of Node or Link or subclass), you can also access the information about the data item from the data provider, so that you bind a skin property directly to a property in the data item. You can access this information through the data property that is part of the Renderer class, for example: 
<s:Label text="{hostComponent.data.finish}"  
  fontSize="10" 
  color.selection="white" 
  color.critical="white" 
  color="black" />
