How to make the diagram reflect the changes in the data source

Both nodeDataProvider and linkDataProvider are of types that extends flash.events.IEventDispatcher. Hence the Diagram can install listeners on those properties to monitor events widely used in Flex, like mx.events.CollectionEvent or mx.events.PropertyChangeEvent. If data nodes or link nodes are added to the data provider properties, or removed, their renderer instances will be removed from the Diagram component accordingly. Installing a filter on an ICollectionView will somehow produce the same result : objects that are filtered out from the data provider are not reachable when parsing the collection, and will consequently not be rendered in the Diagram. Other changes in data providers can be monitored as well : property changes. When a data node for instance has a property called ‘label’ that appears in its renderer, you usually want a value change to be reflected in the Diagram. But for that to happen, the data node Object has to dispatch events when such changes occur. The common way to achieve that is to declare the properties of the data node bindable in Flex sense. For example, here is a class representing a node in the data model exposing a property called name which is declared bindable. Thus, when this property changes, a mx.events.PropertyChangeEvent will be propagated to the collection it belongs to, and this event will be treated by the Diagram component to reflect the changes graphically.
// class representing a node in the data model
public dynamic class DataNode
{
  private var _name:String;
    
  [Bindable]
  public function get name():String
  {
    return _name;
  }
    
    // some other properties...
}
Note : you can also declare the whole class bindable (by putting the [bindable] tag befor the class declaration) for making all its properties bindable.
