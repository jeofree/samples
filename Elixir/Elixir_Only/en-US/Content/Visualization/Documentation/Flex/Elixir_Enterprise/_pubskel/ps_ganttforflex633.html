Accessibility in 3D chart components

The 3D chart components in the following table have built-in support for accessibility: they automate the common accessibility practices of providing text equivalents and promoting keyboard access.
Component Accessibility implementation class
AreaChart3D CartesianChart3DAccImpl
BarChart3D CartesianChart3DAccImpl
ColumnChart3D CartesianChart3DAccImpl
LineChart3D CartesianChart3DAccImpl
PieChart3D PIEChart3DAccImpl
The chart role of MSAA (Microsoft Active Accessibility) does not allow data points to be exposed, so these 3D chart components are exposed through a list role instead. The label of a data point exposed to a screen reader comes from its data tip. And you can customize this tip using custom data tip functions at the chart level or at the series level.
To make navigation through the data points in the chart possible, set the selectionMode property of the chart to single or multiple. The user of the chart can then use the arrow keys to navigate as described in User interaction with 3D charts.
For performance reasons, the data points in the chart are cached by the accessibility implementation. If a change is made to the chart that requires the updating of the cache, call the invalidateDataPoints method on the accessibility implementation of the chart instance as shown in the following ActionScript code.
var accImpl:ChartBaseAccImpl = chart.accessibilityImplementation as ChartBaseAccImpl;
accImpl.invalidateDataPoints();
For more information on accessibility see the section “Advanced Flex Programming>Creating Accessible Applications” in the Flex documentation.
