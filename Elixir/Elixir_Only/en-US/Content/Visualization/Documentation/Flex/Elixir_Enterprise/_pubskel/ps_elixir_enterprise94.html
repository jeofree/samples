Accessibility in calendars

The calendar component has built-in support for accessibility: it automates the common accessibility practices of providing text equivalents and promoting keyboard access.
The accessibility implementation class of a Calendar component is CalendarAccImpl.
The calendar is exposed through a list role of MSAA (Microsoft Active Accessibility). The first child (in the MSAA sense) is the time selection and the others are the events that are in the current visible time range.
The label of an event exposed to a screen reader comes from its data tip. You can customize this tip using a custom data tip function.
The user of the calendar can select time, navigate through displayed events, and edit these events with the keyboard as described in User interaction with calendars.
For more information on accessibility, see Creating Accessible Applications in the Advanced Flex Programming Adobe documentation.
