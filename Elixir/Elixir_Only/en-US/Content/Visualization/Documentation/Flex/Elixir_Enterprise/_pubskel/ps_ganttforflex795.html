Accessibility in radar charts

The radar chart component has built-in support for accessibility: it automates the common accessibility practices of providing text equivalents and promoting keyboard access.
The accessibility implementation class of a radar chart is RadarChartAccImpl.
The chart role of MSAA (Microsoft Active Accessibility) does not allow data points to be exposed, so the radar chart is exposed through a list role instead. The label of a data point exposed to a screen reader comes from its data tip. And you can customize this tip using custom data tip functions at the chart level or at the series level.
To make navigation through the data points in the chart possible, set the selectionMode property of the chart to single or multiple. The user of the chart can then use the arrow keys to navigate as described in User interaction with radar charts.
For performance reasons, the data points in the chart are cached by the accessibility implementation. If a change is made to the chart that requires the updating of the cache, call the invalidateDataPoints method on the accessibility implementation of the chart instance as shown in the following ActionScript code.
var accImpl:ChartBaseAccImpl = chart.accessibilityImplementation as ChartBaseAccImpl;
accImpl.invalidateDataPoints();
For more information on accessibility see the section “Advanced Flex Programming>Creating Accessible Applications” in the Flex documentation.
